<div style="text-align: center; font-style: italic; color: pink;"> <h1>일단 이 방법은 아래 조건에 해당하시는 분만 복구 가능합니다.🥹👍🏻</h1> <ul> <li>commit 을 안하고 pull 이나 rebase 등 파일이 지워진 경우</li> <li><strong>인텔리제이 사용하시는 분</strong></li> <li>그냥 뻘짓하다가 로컬 파일 삭제한 경우</li> </ul> </div>
<p align="center"> 집중해서 일하다가 커밋을 까먹었거나 혹은 깃이 익숙치 않아서 삽질하다가 파일이 다 날라가버렸다던가 뭐 버전관리 하다가 명령어 잘못 쳤다던가..<br>
저도 깃을 다루는데에 왕고수는 아니고 그렇다고 허접은 아니지만 그런 일이 생긴적이 있었습니다. 마감 기한이 다가오기 하루 전 이던가 ^^..<br>
겉으론 아무일 없는 척 했지만 정말 광광 울면서 온갖 키워드를 검색하며 개큰 고통을 겪고 복구 시킨 적이 있었답니다..<br>
뭐 실수로라도 그런 일이 있어서는 안되겠지만<br> 이미 벌어진 이상 빨리 원상복구 시켜야겠죠? 인텔리제이를 이용한 방법과 git 명령어로 살리는 법 </p> <p align="center" style="color: gray;">(이 방법은 노가다에요🤔 최후의 수단입니다.)</p> <p align="center">알려드리겠습니다.</p>

<p align="center" style="font-size: 25px; color: #00FFCC;"><strong>Git 명령어를 사용하는 경우</strong></p>
    <p align="center">
        커밋을 안하고 버전관리를 하는 둥(개노답) 그걸 수습하겠다고 체크아웃하고..  'git log --oneline', 'git reset --soft' 등 이것저것 쳐보다가<br>
        뭐가 뭔지 모르겠고 엄마 보고싶고 눈물나고 그럴때.. 맨 위 해당사항에 인텔리제이 사용 하는 사람에 해당이 안될때..뭘 해도 다 복구가 안될때.. 저는 이 방법으로 복구 시켰습니다.
    </p>
    <p align="center">
        1. 아래의 Git 명령어를 터미널에 입력해주세요.
    </p>

<div style="
  background-color: rgba(128, 128, 128, 0.5);
  padding: 20px; font-size: 20px;
  text-align: center; 
  font-family: 'Roboto', sans-serif;
  width: 50%;
  border: 1px solid white;
  border-radius: 20px;
  margin: 0 auto;
    color: #ffffff;
">
  <strong>
    git fsck --lost-found
  </strong>
</div>

<p align="center">
        2. 그러면 아래 캡쳐본 처럼 dangling blob / dangling Commit / dangling Trees / dangling Tags 넷 중 하나로 시작하는 로그가 보일거에요.<br>
삭제되거나 잘못된 작업으로 인한 파일, 커밋, 디렉토리, 태그를 의미합니다.
<p>
    <p align="center">
        <img width="602" alt="스크린샷 2024-08-12 오전 4 55 08" src="https://github.com/user-attachments/assets/c7e938b1-0ad4-48a4-8656-ac9bab1d6ecf">
    </p>
    <p align="center">
         (실수로 삭제빵 갈겨버린 파일 갯수와 동일하게 나올것입니다..^^ 소근소근)
    </p>
    <p align="center">
        3. 여기서 부터가 조금 노가다입니다. dangling blob 을 제외한 뒤에 랜더믹한 문자열이 파일의 해시값이에요. 해당 해쉬값을 복사한 뒤 아래 명령어 뒤에 입력해주세요.
    </p>

">
  <strong>
    git show 복사한 해쉬값
  </strong>
</div>
    <p align="center">
        그럼 아래의 사진처럼 파일의 내용이 출력됩니다. 이 방법은 노가다에요 정말 삭제된 파일 명이 나오지 않으니까 만약 실수로 날려버린 파일이 10개가 넘어가버린다면..? 끔찍하겠죠?
    </p>

"><strong>
  git fsck --lost-found
</strong>
</div>
    <p align="center">
        3. dangling blob 을 제외한 뒤에 랜더믹한 문자열이 파일의 해시값이에요. 
    </p>

        4. 그럼 짜잔! 아무일도 없었던 것 처럼 원래 디렉토리에 파일이 돌아왔습니다~
    </p>
